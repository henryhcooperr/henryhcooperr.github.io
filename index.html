Featured Project: Chess Implementation

Python Chess Implementation

I developed this chess game to explore complex object-oriented design principles and artificial intelligence concepts. Chess presents unique challenges in software design - while its rules are well-defined, creating a clean, maintainable implementation requires careful consideration of class hierarchies and game state management.

Fundamental Requirements

The game is a full implementation of chess featuring:

• Complete chess rule implementation including special moves

• Support for both human and AI players

• AI opponent using minimax algorithm with alpha-beta pruning

• Intuitive terminal-based interface with Unicode chess pieces

• Comprehensive move validation and game state management

• Visual display of possible moves with clear numbering system

• Tracking of captured pieces

• Check, checkmate, and stalemate detection

Technical Deep Dive

Chess Piece Movement System

The piece movement system is built on abstract and concrete classes:

  • • Abstract Piece Class:
    • - Abstract method get_valid_moves(board): Returns list of valid positions
    • - Concrete method move_to(position): Updates piece location
    • - Position tracking with x,y coordinates
    • - Movement history tracking (has_moved flag)
  • • Concrete Piece Implementations:
    • - Queen: Combines diagonal and straight movements
    • - Rook: Implements straight-line movement in 4 directions
    • - Bishop: Implements diagonal movement in 4 directions
    • - Knight: L-shaped movement with 8 possible positions
    • - King: Single-step movement in all 8 directions
    • - Pawn: Direction-based movement with special first move and promotion

Board Management System

Core functionalities of the board system:

  • • Board State Management:
    • - get_piece_at(position): Retrieves piece at coordinates
    • - get_pieces(color): Returns all pieces of specified color
    • - find_kings_position(color): Locates king for given color
  • • Move Processing:
    • - determine_move_type(): Categorizes moves (capture, castle, etc.)
    • - categorize_moves(): Organizes moves by piece and type
    • - create_move_display(): Generates visual move representation
  • • Display Features:
    • - Unicode piece representation (♔♕♖♗♘♙)
    • - Captured pieces tracking and display
    • - Chess notation conversion (A1, B2, etc.)
    • - Move numbering system for selection

Movement and State Management

Move Class Implementation

The Move class handles movement execution and validation:

  • • Core Attributes:
    • - from_position: Starting coordinates
    • - to_position: Target coordinates
    • - piece_moved: Reference to moving piece
    • - piece_captured: Optional captured piece
  • • Key Methods:
    • - execute(): Performs the move on the board
    • - undo(): Reverts the move for AI analysis
    • - Equality comparison for move validation

Game State Controller

The Game class manages the overall game flow:

  • • State Management:
    • - play_turn(): Plays one round of the game and executes player moves
    • - is_check(): Validates check conditions
    • - is_checkmate(): Determines winning conditions
    • - is_stalemate(): Identifies draw conditions
  • • Player Control:
    • - switch_turn(): Alternates between players
    • - get_valid_moves(): Compiles available moves
    • - Player type management (Human/AI)
